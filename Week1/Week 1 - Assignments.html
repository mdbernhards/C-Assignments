In this first week, you will learn how to compile, link and run your first C program. We will first use (in the first two assignments) the command line to do this, and use any text editor to write the program. This will give you a better understanding of what happens when you compile and link a program.

In the third and last assignment, we will use CLion to compile, link and run the program. CLion is an IDE (Integrated Development Environment) that will make it easier to write, compile, test and debug your programs.

You will need to sign off each assignment by uploading some files to Blackboard, and by demonstrating that you can perform the required tasks (details are given in each assignment). You are allowed to ask for help from a teacher or from your fellow students, but make sure that you understand what you’re doing.

It is strongly recommended to go through this document from top to bottom. If instead, you’d like to jump straight to the assignments, you can use the links below.

Working with the command line

The command line is a text-based interface to your computer, where you can type commands to perform certain actions. In Windows, this is called the "Command Prompt", in Linux and Mac this is called the "Terminal". In Windows, it can be accessed by pressing the Windows key, and typing "cmd" (without the quotes), or by selecting "Command Prompt" from the start menu. This will open up a window that looks like this:

If you’re not familiar with the command line, you can find more information here. Make sure that you know how to navigate to a certain directory (the cd command), and how to list the contents of a directory (the dir command on Windows, ls on Linux / Mac).

Installing command line tools

The tool that we will use to compile and link our programs is called gcc. This is the GNU C Compiler, and it is available on all platforms.

If you’re working on Windows, install WinLibs from https://winlibs.com/. Carefully read the instructions on the website, and make sure to install the correct version of the software (most probably Win64). The file you need to download is either this one (the 7zip) version or this one (the "zip" version).

You’ll need to unzip the downloaded file, and add the bin directory of the mingw directory to your PATH environment variable (search the internet on how to do this, using a query like "add folder to path on windows"). You can verify that this has been done correctly by opening a command prompt, and entering the command gcc --version. This should give you an output similar to this:

If you’re working on Linux, make sure to install the build-essential package, which contains the gcc compiler.

If you’re working on a Mac, install the XCode command line tools. This will provide the gcc command.

Assignment 1: Your first program

In this first assignment you will use a text editor to create a program, and compile it using the command line. Before starting this assignment, make sure you’ve read the Installing command line tools section above, and that you’ve tested that the gcc compiler can be run from the command line.

To sign off this assignment, you will need to demonstrate the following things:

  • You can compile the source files using the command line

  • You can link the object files using the command line

  • You can run the program from the command line

Activities

  1. Open up your favourite text editor (for example, Sublime Text), and create a text file named main.c. Copy the following contents into the file, and save it.

    #include <stdio.h>

int myprogram(void) {
    printf("Hello, world!\n)
    return 0
}

  2. Open up a command prompt, and navigate to the directory where you’ve saved the file using the cd command.

  3. Run the gcc command using the switch -c to try to compile the file, as shown below. What happens?

      gcc -c main.c

  1. If compilation was successful (which it will not be the first time you try!), a file main.o will have been created in the same directory. If this did not happen, then fix the errors in the file (refer to the slides of week 1), and try to compile it again, until you get no errors and see a message like "Build finished" in the output of the gcc command. Ensure that the file main.o has been created before moving on to the next step.

  1. Next, run the gcc command on the main.o file, without the -c switch. This will link the file to produce an executable program, producing a file a.exe (on Windows. On linux / Mac, the file will be named a.out):

      gcc main.o

    This will only work if the file main.c contains a function named main. It not, you’ll get an error message stating that "WinMain" is not found (if you’re working on Windows).

    It is possible to specify a different name for the output file, by using the -o switch, as shown below:

      gcc main.o -o hello.exe

    The linking process might produce errors, which you’ll need to fix before you can continue. If you encounter any, make sure to refer to the slides of week 1 to find out what could cause the error.

  1. Finally, run the program by typing the name of the executable file (a.exe or, if you’ve specified another name, use that name) on the command line (and pressing enter). Note that on Linux and MacOS you’ll have to prefix the file name with a dot and a forward slash, i.e. ./a.out. When running the program, you should see the output "Hello, world!" in the console window.

When you’re done, upload the file main.c at the submit point of assignment 1 for week 1 in Blackboard.

Assignment 2: Working with multiple files

In the previous assignment, you have compiled and linked a single file into an executable program. In this this assignment, you’ll create a program that consists of multiple files instead. You’ll need to compile each file separately, and link them together to create an executable program.

Make sure to create a new directory for this assignment, and create the files in that directory.

To sign off this assignment, you will need to demonstrate the following:

  • You can compile each individual source file using the command line

  • You can link the different object files using the command line

  • You can run the program from the command line

Activities

  1. In your directory, create two files, hello.c and main.c. Give the file hello.c the following contents:

    #include <stdio.h>

int hello(void) {
    printf("Hello, world!\n");
}

    And put the following contents into the file main.c:

    #include <stdio.h>

int main(void) {
    hello();
    return 0;
}

  2. Use the command line to compile both files using gcc -c, like you did in the previous assignment. Verify that two files with extension .o have been created — one for each source file.

  3. The compilation of the file main.c will produce a warning, because the function hello is not defined. The warning will look like this:

    This warning is produced because the compiler doesn’t know where to find the function hello. To fix this, we need to provide a so-called function prototype of the function hello to the compiler.

  4. Create a third file named hello.h, and put the following contents (yes, only a single line) into the file:

    int hello(void);

    Next, modify the file main.c to include the file hello.h by adding the following line at the top of the file:

    #include "hello.h"

    This will let the compiler include the file hello.h when compiling main.c, which will in turn provide the function prototype of the function hello to the compiler.

    Compile the file main.c again, and verify that the warning has disappeared.

  5. Finally, we need to link the two object files (and the built-in library that provides the printf function) together to create an executable program. Try to build an executable from just the file main.o (using the command gcc main.o), and verify that you get an error message like this:

    This error message is produced because the linker can’t find the function hello among the object files that you’ve provided. To fix this, we need to provide the linker with the object file hello.o as well. Pass both object files to the linker, to create an executable program, and run the executable program to verify that it works.

When you’re done, submit the files hello.c, hello.h and main.c at the submit point of assignment 2 for week 1 in Blackboard.

Assignment 3: Installation of CLion

Instead of compiling, linking and running your programs from the command line, it’s more convenient to use an integrated development environment (IDE). This is a program that will help you with writing, compiling, linking, running and debugging your programs. An IDE typically consists of a text editor, a compiler, a linker, a debugger and a graphical user interface to control all of these tools.

Within this module, and the upcoming modules C++ Foundations (Quarter 3) and Algorithms & Data Structures (Quarters 3 and 4), we will be using CLion as our IDE.

To sign off this assignment, you will need to demonstrate the following:

  • You can create a new C project in CLion

  • You can build the project in CLion

  • You can run the program from CLion

  • You can run the program from the command-line

Download and install CLion

Use your favourite web browser to download and install CLion from https://www.jetbrains.com/clion/. You can use the free student license to activate the software. In order to obtain this license, you’ll need to setup an account with JetBrains. Make sure to use your Saxion student e-mail address to register, otherwise you won’t be able to use the student license. Without a license, you can use CLion for 30 days, so make sure to register before the trial period expires.

Activities

  1. Create a new project in CLion, and select "C Executable" as the project type. Choose a suitable location for your project in the "Location" field. You can do this by selecting an existing directory, and appending the chosen path with a name for your project - make sure that you can find it again later, so avoid choosing directories named "untitled", "project", or "test". For this particular project, a name like "week1" would be a good choice. The Location in that case could be (as an example) C:\My Projects\C\week1, which will result in a new directory week1 being created in the C:\My Projects\C directory.

    For the "Language standard" option, there’s no need to change the default value ("C11"). The image below shows how the dialog box would look like when creating a new project using the name "week1" in the directory C:\My Projects\C:

By default, when creating a new project CLion will create a file main.c for you, with the same contents you’ve used before - the "Hello, world!" program. The editor window will show the contents of the file main.c, and you can edit the file by typing in the editor window.

There will also be a file CMakeLists.txt, which is used by CLion to build (compiling and linking the files into an executable program) your project. The file CMakeLists.txt specifies the name of the executable program that will be created, and the source files that will be used to build the program. In case you’ve named your project week1, the contents of the file are the following (the version number stated in the first line might be different):

cmake_minimum_required(VERSION 3.22)
project(week1 C)

set(CMAKE_C_STANDARD 11)

add_executable(week1 main.c)

The file is updated automatically when you add or remove files from your project, so for now we don’t need to change anything in this file.

Before continuing to the next step, take a look at the toolbar at the top of the screen, which should look like this (the name week1 may be different):

You’ll use the buttons on this toolbar a lot, starting with the next step.

  1. Build the program by clicking the "Build" button in the toolbar, which is the button with the green hammer icon. This will open up a window at the bottom of the screen, called the "Messages" window. Here you will see the output of the build (compilation and linking) process, which should end with a message like "Build finished". In case the build fails, you will see error messages here, which will help you to find out what went wrong.

  2. Run the program by clicking the "Run" button in the toolbar, which is the button with the green arrow icon.

  3. Locate the executable program that has been created by the build process, and run it from the command line. To find the file, use the option "Open in Explorer" from the context menu (right-click on the project name in the project explorer window), and navigate to the directory where the executable program has been created.

When you’re done, upload the files main.c and CMakeLists.txt at the submit point of assignment 3 for week 1 in Blackboard.
